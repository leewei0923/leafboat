# _getStyle方法

# 定义一个函数,用来获取指定元素的当前样式
参数: obj 要获取样式的元素
      name 要获取的样式名

function getStyle(obj ,name){
    if(window.getComputedStyle)
    //正常浏览器的方式,具有getComputedStyle
    return getComputedStyle(obj, null)[name];
}else{
    //ie8方式
    return obj.currentStyle[name];
}

# 事件的冒泡(Bubble)
 - 所谓的冒泡指的是时间的向上传导,当后代元素上的事件被触发时,其祖先元素的相同时间也被触发
 - 在我们的开发中冒泡都是有用的,如果不希望发生冒泡,用事件对象取消冒泡
 -取消冒泡:
  - 绑定一个事件对象,event.cancelBubble = true;

# 事件的委派

# 
 - 为每一个超链接绑定一个单击响应函数
 - 这种操作比较麻烦,只能为已有的超链接设置事件,二新添加的超链接必须重新绑定
 - 只绑定一次事件,即可应用到多个元素上,即使元素是后添加的
 - 可以将该元素尝试将其绑定给元素的共同的祖先元素
# 事件的委派
 - 将事件统一绑定给元素的共同祖先元素,这样当后代元素上的事件触发时,会只冒泡到祖先的元素,通过祖先事件的响应函数,来处理事件
 - 事件委派,是利用了冒泡,利用委派,减少事件的绑定次数,提高了性能

# 事件绑定

# 使用事件 对象.事件 = 函数的形式绑定响应函数
 - 它只能同时为一个元素的一个事件绑定元素
 - 不能绑定多个,如果绑定了多个,则后边会覆盖前边的 

# addEventListener()
 - 通过这个方法也可以为元素绑定响应函数
 - 参数:
  - 1.事件的字符串,不要on
  - 2.回调函数,当时间触发的时候会被调用
  - 3.是否在捕获的阶段触发事件,需要一个布尔值,一般false
 - 使用addEventListener()可以同时为一个元素的相同事件同时绑定多个响应函数,这种事件被触发时,响应函数将会按照函数的绑定顺序执行

不支持ie8以下浏览器不支持 ie支持 attachEvent() # attachEvent() - 在ie8中可以使用attachEvent()来绑定事件 - 参数: - 1.事件字符串,on - 2.回调函数 - 这个方法可以为多个时间绑定多个处理文件,不同的是,,它后绑定先执行,执行的顺序和addEventListerner()相反

# 定义一个函数,用来为指定元素绑定响应函数
 - 参数:
   - obj 要绑定事件的对象
   - eventStr 事件的字符串(不要on)
   - callback 回调函数
# 
 - addEventListener()中的this是绑定事件的对象
 - attachEvent()中的this,是window
  - 需要统一两个方法 this

function bind(obj, eventStr, callback){


    if(obj.addEventListener){
    //大部分浏览器兼容的方式
    obj.addEventListener( eventStr, callback ,false);

    
    else{
         //Ie8浏览器
         //this 是谁由调用方式决定
         callback.call(obj)
    obj.attachEvent("on"+ eventStr, function(){
        //在匿名函数中调用callback
        callback.call(obj);
    } );
    }
   
}

<!DOCTYPE html>
<html>
<head>
	<title>拖拽事件</title>
	<style type="text/css">
		#box1{
	        width: 100px;
	        height: 100px;
	        background-color: red;
	        position: absolute;
		}
	</style>
</head>
<body>
	<div id="box1">
		
	</div>
	<script>
		window.onload = function (){
			//拖拽元素box1
			//1.当拖拽元素被鼠标按下拖拽时,随鼠标一起懂 onmousedown
			//2.当鼠标移动时候,还不放手 跟随鼠标移动  onmousemove
			//3.当鼠标松开时,被拖拽元素固定在当前位置 onmouseup


			//获取box1的id
			var box1 = document.getElementById("box1");
			//为box1绑定鼠标按下事件
			box1.onmousedown = function (){


				//当鼠标移动时被拖拽元素跟随元素移动 onmousemove
				//为document绑定一个onmousemove事件
				document.onmousemove = function (event){
					//当鼠标被拖拽时,跟着鼠标移动 onmousemove
					event = event || window.event;
					//获取鼠标的坐标

					var left = event.clientX;
					var top = event.clientY;
					//修改box1的位置
					box1.style.left = left +"px";
					box1.style.top  = top +"px";
				};
				//当鼠标松开时,被拖拽元素固定在当前位置 onmouseup
				//为元素绑定松开元素事件

				box1.onmouseup = function (){
					//取消document的onmusemove事件
					document.onmousemove = null;
					//取消document的onmouseup事件
					document.onmouseup = null;
				}

			};

		}
	</script>
</body>
</html>

# js_基础Navigator

# Bom 浏览器对象模型
 - 在BOM中为我们提供了一组对象,用来完成对浏览器的操作
 - BOM对象
  - window
   - 代表整个浏览器的窗口,同时window也是全局对象
  - Navigator
   - 代表的当前浏览器的信息,通过该对象可以来识别不同的浏览器
  - Location
   - 代表当前浏览器地址栏信息,通过Location可以获取地址栏信息,或者操作浏览器跳转页面
  - History
   - 代表浏览器的历史记录,操作浏览器的历史记录
   - 由于隐私原因,不能获取具体历史记录,只能操作浏览器向后翻页,
   而且该操作只在当次访问有效
  - Screen
   - 代表用户的屏幕信息,通过该对象可以获取到用户的显示器的相关信息
# 这些BOM对象在浏览器中都是作为window对象属性保存,可以通过window对象直接使用,也可以直接使用


# Navigator
 - 代表的当前浏览器的信息,通过该对象可以来识别不同的浏览器
